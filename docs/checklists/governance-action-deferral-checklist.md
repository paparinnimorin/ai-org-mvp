# Governance Action Deferral Checklist

## Pre-check
- [ ] source minutes を特定した
- [ ] 最新action registerを参照した
- [ ] close/reopen/scope-changeで即時解決不能であることを確認した

## Deferral Design
- [ ] deferral理由（dependency / capacity / risk）を記録した
- [ ] due変更（旧→新）を記録した
- [ ] checkpoint deliverable（延期中の最小成果）を設定した
- [ ] resume trigger（再開条件）を定義した
- [ ] KPI impact（遅延コスト）を記録した

## Governance Alignment
- [ ] decision owner と decision due を設定した
- [ ] 次回review dateを設定した
- [ ] action register更新責任者を明記した

## Outputs
- [ ] deferral report を生成した
- [ ] bundle check を実行した
- [ ] `docs/README.md` の関連リンクを更新した
- [ ] `docs/ticket-status.md` を更新した
